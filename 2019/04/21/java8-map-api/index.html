##compute()

等价于

根据已知的 k v 算出新的v并put。
注意：如果无此key，那么oldVal为null，lambda中涉及到oldVal的计算会报空指针。
源码和merge大同小异，就不放了。

##computeIfAbsent()
由上可知，compute()有空指针的风险。所以用computeIfAbsent()来规避。

当key == null 或者不存在时，才compute。该方法看起来有点多余，如果计算不涉及key，可以直接用putIfAbsent()代替。
